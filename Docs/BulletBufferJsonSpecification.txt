目的
----
BulletBuffer 用 JSON を作るときに、BulletDataJson の各フィールドが通常弾と LASER でどう使われるかを整理する。
この仕様は、現在の実装に基づく。


1. BulletBuffer JSON の基本形
---------------------------
{
  "name": "BufferName",
  "bullets": [
    {
      "originPos": { "x": 0, "y": 0 },
      "originVlc": { "x": 0, "y": 0 },
      "playerInfluence": { "x": 0, "y": 0 },
      "startX": 0,
      "speed": 4,
      "gravity": 0,
      "initialAngle": 0,
      "angleSpeed": 0,
      "polarForm": { "x": 1, "y": 0 },
      "radiusVlc": 0,
      "thetaVlc": 0,
      "startPos": { "x": 0, "y": 0 },
      "polynomial": { "x": 0, "y": 0, "z": 0, "w": 0 },
      "typeName": "normal",
      "scale": { "x": 1, "y": 1 },
      "color": { "x": 1, "y": 1, "z": 1, "w": 1 },
      "appearTime": 0,
      "appearDuration": 0,
      "life": 10,
      "random": 0,
      "warpCooldown": 0,
      "unCounterable": false
    }
  ],
  "homing": false,
  "isLaser": false
}

トップレベル:
- name
  BulletBuffer 名。StageData の bulletSpawners[].clipName から参照される。
  空の場合、読み込み元ファイル名から補完される。

- bullets
  BulletDataJson の配列。必須。空配列は実質何も生成しない。

- homing
  true の場合、スポーン位置からプレイヤー位置へ向く角度が、各 bullet の polarForm.y に加算される。
  StageData 側の spawner.angle は無視される。

- isLaser
  false: 通常弾として QuadOrder.enemyBullets に追加される。
  true: LaserEmitter.EmitLASER() に渡され、各 bullet が LASER オブジェクトになる。

読み込み場所:
- 組み込み: Rumia_0, Rumia_1, Line, LineLaser, Circle
- 共通: Assets/BulletBuffers/common/**/*.json
- デバッグ共通: Assets/BulletBuffers/debug/**/*.json
- ステージ別: Assets/BulletBuffers/<stageDirectoryName>/**/*.json
- Mod: mod の BulletBuffers/common, BulletBuffers/debug, または stageData.bulletBufferDirectory

同名 name が既にある場合、後から読まれた BulletBuffer が置き換える。


2. 座標と角度の単位
------------------
- プレイ領域の目安は x: 0..32, y: 0..18。
- JSON 内の polarForm.y, thetaVlc, angleSpeed, initialAngle はラジアン。
- StageData の bulletSpawners[].angle と angleInterval は度数法。
- non-homing の BulletBuffer は、StageData の angle をラジアンに変換して polarForm.y に加算する。
- homing の BulletBuffer は、プレイヤー方向のラジアン角を polarForm.y に加算する。


3. BulletBuffer 生成時の変換
---------------------------
BulletBufferManager.CreateSpawnedBullets() は、JSON から読まれたテンプレート BulletData をコピーしてスポーン用 BulletData を作る。

通常弾・LASER 共通の主な変換:
- spawned.originPos = spawner.pos + rotate(json.originPos, spawnAngleRad)
- spawned.originVlc = spawner.originVlc + rotate(json.originVlc, spawnAngleRad)
- spawned.polarForm.y = json.polarForm.y + spawnAngleRad
- spawned.color = json.color * spawner.color
- spawned.startPos = (startX, polynomial(startX)) + json.startPos

startPos の注意:
- 通常の直線弾で spawn 位置から始めたい場合は、startX = 0, polynomial = {0,0,0,0}, startPos = {0,0} が基本。
- json.startPos は、最終的なローカル座標から引かれる追加オフセットとして効く。
- 例: startX = 0, polynomial = 0, startPos = { "x": -10, "y": 0 } なら、初期位置はローカル +10 方向にずれる。
- 曲線を「曲線上の開始点を原点」として扱いたい場合は、startX と polynomial と startPos の組み合わせを実機で確認すること。BulletBuffer 経由では startPos がそのまま単純代入されず、上記の変換を通る。

色の注意:
- StageData 側の spawner.color と JSON の color は乗算される。
- どちらかが {0,0,0,0} だと不可視になる。無着色で使う側は {1,1,1,1} にする。


4. BulletDataJson フィールド別仕様
--------------------------------
originPos: Vector2
通常弾:
  スポーン位置からの相対原点。BulletBuffer 呼び出し時の角度で回転されてから spawner.pos に加算される。
  originPos は毎フレーム originVlc と playerInfluence で動き、弾の座標計算の基準になる。
LASER:
  BulletBuffer 呼び出し時の角度で回転済みの originPos がレーザー原点として使われる。

originVlc: Vector2
通常弾:
  originPos の移動速度。BulletBuffer 呼び出し時の角度で回転されてから spawner.originVlc に加算される。
  弾道全体を平行移動させる。
LASER:
  LASER の originPos を毎 tick 移動させる速度として使われる。
  StageData の spawner.originVlc と、呼び出し角度で回転済みの JSON originVlc が加算される。

playerInfluence: Vector2
通常弾:
  プレイヤー速度を originPos に反映する倍率。
  {1,0} ならプレイヤーの x 速度に追従、{0,1} なら y 速度に追従する。
LASER:
  使われない。

startX: float
通常弾:
  polynomial 上で計算を始めるローカル x。
  nowCalculateX の初期値になる。
LASER:
  LASER の initialCalculateX になる。speed によってこの x が進む。

speed: float
通常弾:
  polynomial の接線方向に進む速度。
  speed = 0 ならローカル x は進まず、originVlc, thetaVlc, radiusVlc, gravity などでのみ動く。
LASER:
  レーザーの先端が polynomial に沿って伸びる速度。
  静止した直線レーザーを見せたい場合でも、現実装では maxCount の計算に speed が関係するため、基本は正の値を入れる。

gravity: float
通常弾:
  appearTime 後の経過時間 lapse に対して、y 方向へ gravity * lapse^2 / 2 だけ下げる。
  正の値で下へ曲がる。
LASER:
  使われない。

initialAngle: float
通常弾:
  描画時に angle に加算される見た目用の回転。
  移動方向そのものは変えない。
  現在の通常弾衝突判定は bullet.angle を使い、initialAngle は加算しない。
LASER:
  使われない。

angleSpeed: float
通常弾:
  描画角 angle に angleSpeed * lapse を加える。
  見た目を回転させたいときに使う。
LASER:
  使われない。レーザーの回転には thetaVlc を使う。

polarForm: Vector2
通常弾:
  x はローカル弾道の拡大率 r。
  y はローカル弾道の回転角 theta。
  毎フレーム radiusVlc, thetaVlc が加算される。
LASER:
  y だけが初期角として使われる。
  x は LaserEmitter で 1 に置き換えられるため使われない。

radiusVlc: float
通常弾:
  polarForm.x に毎フレーム加算される半径方向速度。
  正なら拡大、負なら縮小。
LASER:
  使われない。

thetaVlc: float
通常弾:
  polarForm.y に毎フレーム加算される角速度。
  弾道全体を回転させる。
LASER:
  LASER の thetaVlc として使われ、レーザーの進行方向を回転させる。

startPos: Vector2
通常弾:
  ローカル座標から引かれる基準点。BulletBuffer 経由では (startX, polynomial(startX)) + json.startPos が実際の startPos になる。
  直線・原点開始なら {0,0}。
  負方向に入れると初期表示位置を正方向へずらせる。
LASER:
  LASER のローカル基準点として使われる。
  BulletBuffer 経由では通常弾と同じく (startX, polynomial(startX)) + json.startPos が渡る。

polynomial: Vector4
通常弾:
  y = x*a + x^2*b + x^3*c + x^4*d の係数。
  x は speed によって進み、曲線の接線から nowCalculateVlc が再計算される。
LASER:
  レーザーの中心線カーブとして使われる。
  通常弾と同じく y = x*a + x^2*b + x^3*c + x^4*d として評価される。
  傾きも通常弾と同じく a + 2*b*x + 3*c*x^2 + 4*d*x^3 を使い、レーザー先端の x 進行量を決める。
  polynomial 上の点列から中心線を作り、その中心線に幅を付けてメッシュを作る。

typeName: string
通常弾:
  BulletTypeDataBase から typeId に解決され、描画テクスチャ、baseSize、renderPriority、衝突形状、カウンター威力に使われる。
  無効な typeName は typeId = -1 になり、通常描画でエラーの原因になる。
LASER:
  現在のレーザー生成では typeId を使わない。
  ただし警告を避けるため、有効な typeName を入れるのが無難。既存レーザーは box を使っている。

scale: Vector2
通常弾:
  描画サイズと衝突形状の倍率。
  実際の描画サイズは scale * BulletType.baseSize。
  ダッシュで消したときのカウンター威力にも max(abs(scale.x), abs(scale.y)) が掛かる。
  warp_zone の場合は、ワープ判定矩形の幅・高さになる。
LASER:
  max(abs(scale.x), abs(scale.y)) がレーザー長になる。
  scale.y は幅としては使われない。

color: Vector4
通常弾:
  描画色。StageData の spawner.color と乗算される。
  w は透明度。
LASER:
  LASER マテリアル色。StageData の spawner.color と乗算済みの値が渡る。

appearTime: float
通常弾:
  出現完了時刻。time < appearTime の間は衝突判定されず、appearDuration に応じて描画も抑制される。
  その間、弾道更新は dt * 0.00001 程度でほぼ止まる。
LASER:
  出現遅延ではなく、レーザー幅 width として使われる。
  見えるレーザーにするには正の値を入れる。

appearDuration: float
通常弾:
  appearTime 直前の予告表示時間。
  > 0 の場合、appearTime - appearDuration から appearTime までビートに合わせた薄い表示になる。
  0 の場合、appearTime までは不可視。
LASER:
  使われない。

life: float
通常弾:
  life > 0 なら time >= life で非アクティブになる。
  life <= 0 は時間による消滅なし。
  描画では life 直前 0.1 秒でフェードアウトする。
LASER:
  LASER の残り寿命。毎フレーム減り、0 未満で破棄される。
  0 以下だとほぼ即座に消えるので、レーザーでは正の値を推奨。

random: float
通常弾:
  originPos に滑らかなノイズを加える強度。
  0 ならノイズなし。
LASER:
  使われない。

warpCooldown: float
通常弾:
  ワープ直後の再ワープ防止時間。毎フレーム減る。
  JSON で初期値を入れると、開始直後のワープを遅らせられる。
LASER:
  使われない。

unCounterable: bool
通常弾:
  true の弾は、プレイヤーダッシュ中でもカウンターで消せない。
LASER:
  使われない。LASER は通常弾のカウンター処理を通らない。


5. BulletData のランタイム専用フィールド
--------------------------------------
以下は BulletData には存在するが、BulletBuffer JSON では直接書かない、または書いても BulletDataJson からは読まれない。

position:
  現在位置。スポーン直後は spawner.pos から始まり、BulletDataUpdateJob または LASER 内部更新で変化する。

velocity:
  通常弾の前フレームからの変位。角度計算、カウンター弾生成、ワープ後の速度復元に使う。
  単位時間あたり速度ではなく、通常更新では「今回の移動量」として入る。

angle:
  通常弾の現在角度。BulletDataUpdateJob が velocity から求める。
  描画では angle + initialAngle、衝突判定では angle が使われる。

nowCalculateVlc:
  polynomial 接線方向の現在ローカル速度。BulletBuffer のコピー時に startX と polynomial から計算され、毎フレーム更新される。

nowCalculateX:
  polynomial 上の現在 x。startX から始まる。

typeId:
  typeName から解決された整数 ID。通常弾の描画・衝突・特殊種別判定に使う。

areaNum:
  現在位置が属する Morton セル番号。衝突判定用の空間分割に使う。

time:
  生成後の経過時間。appearTime, life, angleSpeed, gravity, random に関係する。

isActive:
  有効フラグ。通常弾は範囲外・寿命切れ・カウンター消滅などで false になる。

isClearing, clearTime, clearDuration:
  ClearManagedEnemyDanmaku() などで弾幕を消すときのフェード用。


6. 通常弾の更新式
----------------
毎フレーム BulletDataUpdateJob が通常弾を更新する。

基本:
- time += dt
- warpCooldown -= dt
- life > 0 かつ time >= life なら isActive = false
- time < appearTime の間は、ほぼ止まった状態で更新され、衝突判定されない

原点:
- originPos += originVlc * dt
- originPos += playerVelocity * playerInfluence * dt
- random > 0 なら originPos に滑らかなノイズを加える

ローカル曲線:
- x = nowCalculateX + nowCalculateVlc.x * dt
- y = polynomial.x*x + polynomial.y*x^2 + polynomial.z*x^3 + polynomial.w*x^4
- disVector = (x,y) - startPos
- nowCalculateVlc = normalize((1, polynomial'(x))) * speed

極座標変換:
- polarForm.x += radiusVlc * dt
- polarForm.y += thetaVlc * dt
- rotatedVector = polarForm.x * rotate(disVector, polarForm.y)
- position = rotatedVector + originPos + noise - gravityEffect

角度:
- angle = atan2(velocity.y, velocity.x) + angleSpeed * (time - appearTime)
- 描画角は angle + initialAngle

範囲:
- position.x < 0, position.y < 0, またはセル範囲外になると isActive = false


7. LASER の生成・更新
--------------------
isLaser: true の BulletBuffer は、通常弾リストには追加されず、LaserEmitter.EmitLASER(List<BulletData>, spawner.pos) に渡る。
bullets 配列の各要素が 1 本の LASER になる。

BulletData から LASER への対応:
- spawned originPos -> LASER originPos
- originVlc -> LASER origin velocity
- thetaVlc -> LASER angular velocity
- speed -> レーザー先端の進行速度
- polarForm.y -> 初期角度
- startX -> initialCalculateX
- startPos -> レーザー中心線の基準点
- polynomial -> レーザー中心線の曲線
- max(abs(scale.x), abs(scale.y)) -> レーザー長
- appearTime -> レーザー幅
- life -> レーザー寿命
- color -> LASER マテリアル色

LASER 中心線の更新式:
- internal dt = 0.03
- originPos += originVlc * dt
- theta += thetaVlc * dt
- slope = polynomial.x + 2*polynomial.y*x + 3*polynomial.z*x^2 + 4*polynomial.w*x^3
- x += speed * dt / sqrt(1 + slope^2)
- y = polynomial.x*x + polynomial.y*x^2 + polynomial.z*x^3 + polynomial.w*x^4
- local = (x, y) - startPos
- point = originPos + rotate(local, theta)

LASER で無視される主なフィールド:
- originPos
- playerInfluence
- gravity
- initialAngle
- angleSpeed
- polarForm.x
- radiusVlc
- typeName/typeId
- appearDuration
- random
- warpCooldown
- unCounterable

LASER 制作時の注意:
- appearTime は遅延ではなく幅。幅 0 はほぼ見えない。
- scale は長さ。幅ではない。
- 静止した太い線を作るつもりでも、現実装では speed や thetaVlc/startPos の関係でメッシュ点数 maxCount が決まる。
- 直線レーザーは、まず speed > 0, polynomial = 0, startX = 0, startPos = {0,0}, positive appearTime, positive life で作るのが安全。
- LASER は通常弾の typeId 描画・衝突・カウンター処理を通らない。


8. 特殊 typeName
----------------
有効な通常 BulletType の例:
- anchor
- attention
- box
- chain
- hummer
- knife
- normal
- simpleline
- slash
- splash_0
- splash_1
- splash_2
- sword
- tear
- warp_zone

注意:
- rocket.asset は存在するが typeName が空なので、JSON では使わない。

warp_zone:
- 通常弾として生成しようとすると、enemyBullets ではなく warpZones に入る。
- scale がワープ矩形のサイズになる。
- 2 個以上置くと、zoneIndex と zoneIndex ^ 1 の組み合わせで相互ワープする。
- ワープした通常弾は、位置・originPos・originVlc が出口に合わせて再設定され、speed, gravity, radiusVlc, thetaVlc, playerInfluence, random, polynomial などがリセットされる。

ScreenNoise:
- BulletTypeDataBase ではなく BulletData.ResolveTypeId() が特別に ScreenNoiseTypeId = -1000 を返す。
- 生成時に CManager.StartScreenNoise() に渡され、通常弾としては追加されない。
- appearTime は開始遅延、life は終了時刻、scale は画面ノイズの強度に使われる。
- color, polynomial, speed などは実質使われない。


9. 作成パターン
---------------
直線通常弾:
- isLaser = false
- startX = 0
- speed > 0
- gravity = 0
- polarForm = { x: 1, y: 0 }
- radiusVlc = 0
- thetaVlc = 0
- startPos = { x: 0, y: 0 }
- polynomial = { x: 0, y: 0, z: 0, w: 0 }
- appearTime = 0
- life > 0

リング・全方向弾:
- 複数 bullet を用意する。
- polarForm.y = 2*pi*i/n。
- speed > 0。
- thetaVlc を入れると回転しながら広がる。

曲線弾:
- polynomial に係数を入れる。
- speed は曲線接線方向の速度。
- polarForm.y で曲線全体を回転する。
- startPos の影響が強いので、まず {0,0} で確認してからオフセットを入れる。

遅延出現:
- 通常弾では appearTime に出現完了時刻を入れる。
- 予告表示したい場合は appearDuration > 0。
- 衝突判定は time >= appearTime まで発生しない。

直線 LASER:
- isLaser = true
- bullets 1 個につき LASER 1 本。
- speed > 0
- polarForm.y に角度を入れる。
- scale に長さを入れる。例: { x: 18, y: 1 }
- appearTime に幅を入れる。例: 0.15
- life > 0
- polynomial = 0
- startX = 0
- startPos = {0,0}

回転 LASER:
- isLaser = true
- thetaVlc に角速度を入れる。
- polarForm.y に初期角を入れる。
- life で回転時間を決める。
- 幅は appearTime、長さは scale。

11. チェックリスト
-----------------
- name と StageData の clipName が一致している。
- bullets が 1 個以上ある。
- 通常弾の typeName は BulletTypeDataBase に存在する。
- JSON 内の角度はラジアン、StageData spawner の角度は度数法。
- color と spawner.color のどちらかがゼロになっていない。
- Set `scale` explicitly. Do not use `size`.
- 通常弾は life > 0 で掃除されるようにする。
- 通常弾の startX, polynomial, startPos を意図した初期位置になるよう確認する。
- 通常弾の appearTime は出現遅延、LASER の appearTime は幅。
- LASER は scale が長さ、appearTime が幅、life が寿命。
- LASER で gravity, angleSpeed, radiusVlc, random などを入れても効果はない。
- 画面外や負座標に即座に出る弾は、セル範囲外として非アクティブになる。
