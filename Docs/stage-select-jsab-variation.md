# ステージ選択 JSAB バリエーション 設計

参考: `Instructions/jsab stage-select.PNG`(JSAB チャプター選択画面)
方針: 既存のステージ選択(縦リスト)とは**別のバリエーション**として共存させ、選択画面上でスタイルを切り替えられるようにする。

## 参考画像の構成(再現対象)

- 上部: シアンの横帯ヘッダー(♪アイコン+「チャプター選択」)。本実装では「ステージ選択」
- 中央: 大きなステージカード(プレビューアート、シアン系の枠)。カード上部にステージ名
- 左右: 前後ステージのカードが暗く見切れて配置(境界に縦ライン)。横送りのキーヒント
- カード直下: **主人公アイコン → 破線 → ゴール(丸)** の進行インジケータ。主人公は Unity 本編のプレイヤースプライト(ドット絵)を使用
- 下部: 操作ヒントバー(決定 / 戻る 等、実際のキーバインドに合わせる)
- 配色: 黒背景+シアン。JSAB らしいフラット+本作のドット絵素材

## 実装方針

### 統合ポイント(既存フローを壊さない)
- `StageSelectManager` の `State.Music` フェーズの**表示だけ**を差し替える。入力(up/down/button/back)の意味は 横送り(左右=ステージ移動)に読み替え
- 決定時は既存の `TransitionToDifficulty()` に合流し、難易度選択〜`StartGameTransition` は現行のまま
- スタイル切替: 選択画面(Musicフェーズ)で `V` キー。`PlayerPrefs "stageSelectStyle"`(0=現行 / 1=JSAB)で永続化。既定は 0(現行)

### 画面構築
- 新スクリプト `Assets/Scripts/UI/JsabStageSelect.cs` が**ランタイムで自前の Canvas を生成**(シーンへの恒久的な UI 追加はしない。シーン差分を最小化し、既存 Canvas 群と独立)
- ソートオーダーは既存 StageCanvas より上、PixelTransition より下
- ステージ一覧は `GManager.Control.SDB`(StageDataManager)から取得

### プレビューカード
- 中央カード: `VideoPlayer`(url = `Application.dataPath + "/StageData/{stage}/{stage}.mp4"`)→ RenderTexture → RawImage。ループ・ミュート再生
- mp4 が無いステージ: 濃紺フラットカード+ステージ名の大きな表示でフォールバック
- 左右の隣接カード: 静的な暗いカード(動画は中央のみ。負荷と見た目の焦点を両立)。alpha ~0.35
- カード切替時: 中央カードのスライド+フェード(0.25s 程度)、動画 url 差し替え

### 進行インジケータ
- 主人公アイコン: 本編プレイヤーの SpriteRenderer からスプライトを取得して UI Image に表示(ドット絵のまま、フィルタ Point、拡大は整数倍)
- 破線+終点の丸(リング)。選択中ステージのクリア状況が取れる場合は反映、無ければ装飾として表示

## 検証
- Play Mode で V 切替 → 横送り → 決定 → 難易度 → ステージ開始まで通し確認
- 現行スタイルが既定のまま無変化であること
- コンソールエラーなし
