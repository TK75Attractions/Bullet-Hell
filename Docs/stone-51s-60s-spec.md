問題: 大ブロックが 47.917s に出て **~5.4秒間 静止**したまま置かれ、52.5s ハンマー→53.333s 破裂。
この静止時間（≈48〜53s、ユーザーの言う「51s の間延び」）を削るため、**破壊を ~51s の実在拍に前倒しし、
ハンマー機構を廃して直接爆破に置き換える**（ユーザー回答済み）。

### アンカー拍

51.0 に最も近い実在拍は **31:3 = 50.833s**（誤差0.167s）。beat3 の中拍アクセント。
代替: **32:1 = 51.667s**（次の小節頭・強拍。爆破の「重い一撃」を強拍に置きたい場合はこちら、誤差0.667s）。
**推奨は 31:3**（51.0 に最も近い）。Play/Oracle で強拍感が欲しいと判断したら 32:1 に上げる。

### chart 編集（`stone.chart.json`）

1. **ハンマー削除**: `石工大ブロックハンマー_3`（`at:"32:3"`）のイベントを丸ごと削除。
   - `_comment`（「33:1のerase_flash_2着弾に合わせる」）も一緒に消える。
2. **破裂を前倒し**: 以下2件の `at` を `"33:1"` → **`"31:3"`** に変更。
   - `石工大ブロック破片_3`（thin easy=2 は維持）
   - `石工大ブロック破裂フラッシュ_3`
3. **要判断（Play/Oracle で確定）**: 次ブロックE群（`石工ブロック予告_E`@32:3 / `石工ブロック出現_E`@33:1 /
   `石工消去フラッシュ_2`@33:1）は**現状の拍のまま据え置きを推奨**（次フレーズの音ハメアンカー）。
   破壊 50.833s → E出現 53.333s の 2.5s 間隔が「間延び」に見えるなら、E群を1フレーズ前倒し
   （予告 31:1=50.0 / 出現 31:3=50.833 など）で詰める案を Play で試す。まず据え置きで挙動を見てから判断。

### 重要な検証点（Play で必ず確認・追加編集が要るかも）

- **大ブロック本体の消え方**: `big_block_spawn_3.json` の弾 `life` が「53.3s のハンマー着弾で消える」前提だと、
  破裂を 50.833s に前倒ししても**本体が 50.833〜53.3s の間 残り、爆破後もブロックが居座る**視覚矛盾が出る。
  - 対処: `big_block_spawn_3` の弾 life を **50.833s 終了**に合わせて短縮する（出現 47.917s ＝相対 t0 とし、
    life ≈ 50.833−47.917 = **2.917**、実 life は当該 buffer の appearTime を差し引いて算出）。
  - もしブロック消去が `石工消去フラッシュ_2`(33:1) や別の erase 機構に依存しているなら、その erase も 31:3 へ。
  - **次セッションは big_block_spawn_3.json と erase_flash_2 の中身/参照を先に確認**してから life・erase を確定する。
- 破裂と本体消滅が **50.833s の同一拍で一致**して見えること（爆破＝消滅の因果）。
- 音ハメ非破壊: 41:1/45:1 等 他フレーズの拍は不変。移動するのは group3 破壊の3イベントのみ。

## (4) 60s 着地スタック解消（縦積み廃止・落下交差も解消）

### 現状（現物・mass_drop_3 / mass_settle_3）

`石工一斉落下_3`（`mass_drop_3.json`、36:4=59.583s、9弾）は全弾 `speed:0`・重力のみ・`life:0.833333`。
**speed0 なので落下中 x は一定＝originPos.x**。着地y = originY − gravity×0.347222（life²/2=0.347222）。

着地は `石工着地_3`（`mass_settle_3.json`、appearTime0.833333）が示すとおり **2段のスタック地形**:

- 上段 y4.7: big(8,4.7) と big(24,4.7)（scale3）
- 下段 y2.3: small×7 = x{3.6, 8.4, 11.4, 16.6, 20.4, 24, 28.2}

→ **x≈8 で big(4.7) が small(8.4,2.3) の上、x=24 で big(4.7) が small(24,2.3) の真上**＝意図的スタック。
落下中も、同一 x 列を共有する 2ブロック（x≈8 ペア・x=24 ペア）だけが交差・通り抜ける。

### 設計（単層化）— スタックと落下交差を同時に解消

**speed0・重力のみ＝各ブロックは自分の x 列を真下に落ちるだけ**。よって
**9個を全て別 x 列（重なり半幅ぶん離す）にすれば、縦スタックも落下交差も同時に消える**。
2つの big を下段 y2.3 へ降ろし、9個を単層の横一列（全て着地 y=2.3）に並べる。

- 重力は**各弾とも不変**（＝落下スピード感・音ハメを保つ）。着地yを 2.3 にするため originY を調整
  （originY = 2.3 + gravity×0.347222）。small7個は元々 y2.3 着地なので originY 不変、x のみ変更。
  big2個だけ originY も変更（落下距離は元と同じ 5.3 / 7.3 ＝ 見た目の落下スピードは不変）。
- 列間隔: big(scale3, 半幅1.5)・small(scale1.8, 半幅0.9)。隣接中心距離を b-s≥2.4 / s-s≥1.8 以上確保。
