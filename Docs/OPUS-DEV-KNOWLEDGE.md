# Opus 開発ナレッジ（Bullet-Hell）

作成: 2026-07-07 Fable。第13〜36便（石工レビュー約60件+タイトル/ステージ選択刷新）で得た知見の総まとめ。
新しいセッションはまず CLAUDE.md → このファイル → PROGRESS.md 直近数便、の順で読むと事故が減る。

## 1. 事故を防ぐ絶対ルール

1. **背景ジョブ/watcher/完了通知待ちをしない**（-p 実行では通知が来ない。録画等の長時間処理は自前ポーリングで sleep しつつファイルサイズ安定を確認）
2. **重い操作（EditMode テスト・録画）の前に仮コミット**（Unity は EditMode テスト中にクラッシュした実績がある）
3. 弾幕データを触ったら: Validate All Stages 0 error → EditMode 全緑 → golden 再生成は**意図したバッファのみ** → BOM+CRLF を byte 検査
4. push 禁止（親/ユーザー判断）。`git add .` 禁止・狭い add。Instructions/ は gitignore
5. 録画は毎回「作ってある部分全体」を音声付きで。**撮影前に stageSelectStyle=0 へ、撮影後 1 に復元**（JSAB 型のまま撮ると箱型プレビューで撮れてしまう）

## 2. 弾幕バッファ系の既知メカニズム（レビュー頻出の根因）

- **ハンドオフの継ぎ目は2種類**あり、どちらも「一瞬消える」に見える:
  - (a) 発火時刻の GAP（例: spawn→drop に1拍 0.417s の隙間）
  - (b) **fresh-fire 1フレーム継ぎ目**: `appearTime==0` で新規発火するクリップは初回スポーンが約1ゲームフレーム遅れるが、退場側は `time>=life` で即消滅する → 1フレームだけ黒くなる。対策は退場側 life に微小余白（+0.034〜0.05s）。事前発火済み（appearTime>0）のクリップには起きない
  - 機械検査: `Tools/audit_handoff.py`（両方を検出できるよう拡張済み。新しいハンドオフを作ったら必ず通す）
- **破片の途中消滅**は `Tools/audit_shard_life.py` で全数棚卸しできる（life と軌道から早期消滅を検出）
- **engine の gravity は -Y のみ**。任意方向の「重力的な加速」は `radiusVlc`（∝t² の radial 加速）で作る（例: 縁カッター破片を壁の外向きに加速して画面外へ。`Tools/gen_edge_cutter_shards.py` 参照）
- コンベア模様は `_StoneBeltScroll` uniform + `StoneBeltScrollDriver`（flow 窓の間だけスクロール）。**belt_flow を retime したらドライバの窓定数も追従必須**（コメント明記済み）
- 予告↔実位置は機械照合 d=0 を維持。warn/spawn/drop/settle/belt を単一定義から再生成して位置ロックする手法が安全（過去のポップ根治で確立）
- ユーザーの感覚値の相場: ハンマー回転 7、破片は「派手だが避けられる密度・小さめ・中央まで飛ばさない・軌跡や刃の位置から発生」、落下は真下・接地・タイル状ランダム（CLAUDE.md の定義参照）

## 3. UI/画面系の設計と罠

### ステージ選択（JsabStageSelect.cs・style 1）
- 5スロットの帯（画面外L・L・C・R・画面外R）+ 3枚パネルのプール役割ローテーション。遷移はパネル自体の位置+スケール同時トゥイーン（0.3s ease-out）、新パネルは画面外(±1220)からスライドイン
- 中央枠はチャンファ+グローの手続き生成テクスチャを 9 スライスで（サイズ変化で角が崩れない）
- 上部バーは style 0 の `Head` を実行時 clone し毎フレームミラー（タイマー・赤ゲージ・警告パルスも従来と同一になる）
- **タイマーが止まって見える/タイトルが飛ばされる時**: `RefreshStyleVisibility` と GManager の Title 状態の関係を疑う（JSAB は不透明フルスクリーンキャンバスなので、出すタイミングを誤るとタイトルを覆い隠す）
- フラッシュ/ポップの定番原因: ①rootCG.alpha を下げると背後のゲーム画面が透けて「発光」に見える ②RenderTexture 準備前の表示（ready フラグで保留する）③CanvasGroup 中間 alpha で「全面塗り+暗 body」構造の枠が透けて光る（枠は4辺の帯=リム方式にする）④dt スパイクでフェードが1フレームに潰れる（dt を 1/30s にクランプ）⑤ease-out の初速が速すぎて2フレームで急伸（線形にする）

### 遷移演出
- **PixelTransition は ScreenSpaceOverlay のルートに置かないと JSAB キャンバスを覆えない**（`EnsureTopmost` で自動移設済み）。`overrideSorting` のネスト Canvas は `isRootCanvas=true` を返す罠あり
- プレイ開始はカバーもリビールも「中央発」: 白ピクセルブロックが中央から広がって覆う → 全白（短く 0.05s）→ 中央から解像。**均一 alpha のフラッシュにすると3回再指摘された**
- タイトルのスタート演出: 選択バナー白フラッシュ（文字はネイビー反転で可読維持）→行が右へ加速スライドアウト→ロゴ上抜け→ステージ選択が交差フェード（0.6s×1.25）

### テキスト・ぼかし・撮影
- **TMP の上下中央合わせ（CenterInkVertically）は非アクティブ時に空振りする**（メッシュ未生成）。表示後の初回フレームで実行。検証は `Tools/measure_vcenter.py`（上下余白の実ピクセル測定）
- 背景ぼかしは `BackdropBlurUtil`（2x ずつのピラミッド縮小+1/4解像度保持）に統一。一括 1/12 縮小はジャギーが出る
- スクショの罠: EditorApplication.update フック撮影は Game View 状態次第で白/黒になる → **Play 内コルーチン+WaitForEndOfFrame**。ScreenCapture は連写時に数フレーム古い内容を書くことがある。連写は `Time.captureFramerate=60`、スライド検証は timeScale 0.1〜0.15 のスロー連写が有効
- 録画は Start+Record の一括（t0 atomic）でないと冒頭 2.6s 前後が欠ける

### デザイン言語（ユーザー承認済み）
- 目標モックアップ: `Instructions\stage-select-mockup.png`（ChatGPT 生成・ユーザー承認）。参考: `Instructions\jsab stage-select.PNG`
- トップバー/ブランド面 = 青グラデ #4290DB、操作アクセント = シアン #38C2E0（チップは青地+白文字）、斜めバナー+白スラッシュブラケット、暗紺背景+幾何学図形（回転四角/SoftCircle/三角+ShapeDrifter、alpha 1/3 程度）
- 引き継ぎ画面はミニマル方針: 無枠の暗パネル+タイポと余白、コード帯と入力行は同一カラム幅、キー操作ヒントは置かない
- 引き継ぎコードは v2=4文字（合計値 p6+c6+CRC8）。旧16文字の読み込み互換は維持（長さで判別）
