# ServTechSlash / Bullet-Hell 配信準備計画

作成日: 2026-06-11  
目標: 2026年10月ごろに無料配信。主目標日は 2026-10-09（金）、予備日は 2026-10-16（金）。

## 前提

- Unity製PCゲームとして、まず Windows 64-bit 版を主配布物にする。
- 無料頒布を基本にする。Steamは無料配信でも Steam Direct のアプリ費用が必要。
- Steam、DLsiteを主軸にし、BOOTH / itch.ioを無料配布の補助導線として使う。
- WebGL版が安定するなら unityroom も候補にする。PCダウンロード版のみなら優先度は下げる。
- プロジェクト設定には `companyName: DefaultCompany`、`productName: ServTechSlash`、`applicationIdentifier: com.DefaultCompany.2D-URP` が残っているため、公開前に正式名称へ更新する。

## 重要な権利確認

リポジトリ内に `Rumia` など東方Project由来に見える名称がある。東方Project二次創作として頒布する場合、東方Projectよもやまニュースの二次創作ガイドラインに従う。

- 二次創作であることを明記する。
- 公式コンテンツと誤認させない。
- 原作ゲームから抽出した素材は使わない。
- 他者の二次創作物を無断利用しない。
- 配布先は、原作が販売されている流通、許可された流通、同人ショップ、インディーゲームサイト等の範囲で考える。

二次創作でない場合も、仮素材、音楽、効果音、フォント、外部アセット、Unity Asset Store素材の再配布条件を一覧化しておく。

参考: https://touhou-project.news/guideline/

## サイト別の手順

### Steam

向いている用途: もっとも広い到達範囲。無料ゲームでもライブラリ登録、レビュー、コミュニティ、実績、アップデート配信が使える。

手順:

1. Steamworks Developer Program に登録する。
2. デジタル書類、銀行情報、税務情報、本人/事業者確認を完了する。
3. Steam Direct Fee をアプリごとに支払う。公式情報では 1アプリあたり 100 USD。
4. アプリ作成時に無料製品として設定する。
5. ストアページを作る。カプセル画像、スクリーンショット、トレーラー、説明文、対応OS、言語、成人向け/年齢関連アンケートを設定する。
6. ストアページをレビュー提出する。Steam公式は通常 3〜5営業日、差し戻しを考えて 7営業日前以上を推奨。
7. ストアページ承認後、Coming Soonページを最低2週間公開する。
8. SteamPipeでビルドをアップロードし、デポ/ブランチ/起動オプションを設定する。
9. ほぼ最終ビルドをレビュー提出する。公式は通常 3〜5営業日、7営業日以上の余裕を推奨。
10. 承認後、希望時刻に Steamworks の Release App から手動リリースする。

注意:

- 初回タイトル等は、アプリ費用支払いからリリースまで30日待機が必要。
- ストアページをビルドより先にレビュー提出する必要がある。
- Coming Soonページは最低2週間必要。
- 無料ゲームでもマーケティング、ウィッシュリスト、ストア文言、スクリーンショット品質が重要。

参考:

- https://partner.steamgames.com/steamdirect
- https://partner.steamgames.com/doc/gettingstarted/appfee
- https://partner.steamgames.com/doc/store/freetoplay
- https://partner.steamgames.com/doc/store/releasing
- https://partner.steamgames.com/doc/store/review_process

### DLsite

向いている用途: 国内同人・二次元ゲームのユーザーに届きやすい。無料ゲーム枠も存在する。

手順:

1. viviON IDを用意し、DLsiteのサークル登録を行う。
2. サークル情報、代表者情報、支払先情報を入力する。
3. 作品登録で、同人ゲーム/フリーゲームとして作品情報を作る。
4. Windows向けZIP、紹介文、スクリーンショット、年齢区分、ジャンル、タグ、動作環境を登録する。
5. 無料配信の場合、0円/無料作品として登録可能かサークル管理画面で確認する。無料ゲームページには 0円作品が掲載されているため枠自体は存在するが、登録条件は早めに確認する。
6. 作品データチェック/審査に提出する。
7. 承認後、指定日公開または公開開始を行う。

注意:

- DLsiteサークル登録ページでは、登録料・利用料無料、データチェックは審査チームが行う旨が案内されている。
- 価格・卸価格表は有料販売向けなので、無料頒布時の設定可否は実際の作品登録画面または問い合わせで確認する。
- 予告作品登録が無料ゲームで使えるかも早めに確認する。
- 成人向け、AI生成、一部AI利用、スマホゲーム等は扱いが分かれるため、該当する場合は別途確認する。

参考:

- https://www.dlsite.com/home/circle/invite
- https://www.dlsite.com/home/genres/works?genrekey=freegame
- https://www.youtube.com/watch?v=-591d1Q6E50

### BOOTH

向いている用途: 国内向けのシンプルな配布置き場。SNSから直接誘導しやすい。

手順:

1. pixivアカウントでBOOTHショップを作る。
2. ダウンロード商品を作成する。
3. ZIP、説明文、スクリーンショット、動作環境、注意事項を登録する。
4. 価格を0円にして無料配布する。
5. 公開後、Steam/DLsite/公式SNSへの導線を整える。

注意:

- BOOTH公式ヘルプでは、ダウンロード商品のサービス利用料は有料時に商品価格 x 5.6% + 45円、無料配布ではサービス利用料なし。
- 審査による公開遅延は比較的少ない想定だが、規約・権利表記は必ず整える。

参考: https://booth.pixiv.help/hc/ja/articles/115004576874

### itch.io

向いている用途: 海外向け、英語ページ、無料/投げ銭/Pay what you want配布。

手順:

1. itch.ioアカウントを作る。
2. New projectでゲームページを作成する。
3. Windows ZIPをアップロードする。WebGL版があるならHTML版としてブラウザプレイも検討する。
4. 価格を無料またはPay what you wantにする。
5. Draft/Restrictedで事前確認し、公開する。

注意:

- itch.io公式FAQでは、ページ作成・アップロードは無料で、収益分配率を作者側で選べる。
- 日本語だけだと見つかりにくいので、最低限の英語タイトル、説明、操作方法、スクリーンショットを用意する。

参考:

- https://itch.io/docs/creators/faq
- https://itch.io/docs/creators/pricing

### その他候補

- ふりーむ！: 国内フリーゲーム導線。審査あり。二次創作や表現内容の扱いは慎重に確認する。https://www.freem.ne.jp/
- unityroom: Unity WebGL版が安定する場合に有効。ブラウザだけで無料プレイできる導線。https://unityroom.com/
- フリーゲーム夢現: 国内フリーゲーム紹介・レビュー導線。PCダウンロード版向けの補助候補。https://freegame-mugen.jp/

## 逆算スケジュール

### 2026-06-11〜2026-06-21: 配信方針と権利確認

- 配信タイトル、サークル/開発者名、ロゴ表記を確定する。
- ProjectSettingsの `DefaultCompany`、アプリID、バージョン表記を正式化する。
- 東方Project二次創作か、完全オリジナルかを明確化する。
- 使用素材台帳を作る。画像、音楽、効果音、フォント、アセット、ライブラリのライセンスを確認する。
- Steam、DLsite、BOOTH、itch.io、必要ならunityroom/ふりーむ！の配信優先度を確定する。

成果物:

- 素材ライセンス表
- ストア掲載名
- 短いゲーム紹介文
- 対応プラットフォーム方針

### 2026-06-22〜2026-07-07: リリース品質の基準決め

- 最低限の完成条件を決める。例: 全ステージ通しプレイ可能、クラッシュなし、設定保存、入力説明、終了導線。
- Windows 64-bit Releaseビルドを作り、別PCで起動確認する。
- キーボード/ゲームパッド対応、解像度、フルスクリーン切替、音量、ポーズ、リトライ導線を確認する。
- 既知の不具合をP0/P1/P2に分類し、10月までに直す範囲を絞る。

成果物:

- QAチェックリスト
- 既知不具合リスト
- 初回テストビルド

### 2026-07-08〜2026-07-21: ストア素材制作

- スクリーンショットを6〜10枚撮る。Steamは実ゲーム画面中心。
- 30〜60秒の短いトレーラーを作る。
- Steam用カプセル画像、ヘッダー、ライブラリ画像を作る。
- DLsite/BOOTH/itch.io用のサムネイルと紹介画像を作る。
- 日本語紹介文、英語紹介文、操作方法、動作環境、配信/実況可否ポリシーを書く。

成果物:

- ストア画像一式
- トレーラー
- JP/ENストア文
- 実況・配信ガイド

### 2026-07-22〜2026-07-31: アカウント登録と初期申請

- Steamworks登録、銀行/税務/本人確認、Steam Direct Fee支払いを完了する。
- DLsiteサークル登録を完了する。
- BOOTHショップ、itch.ioアカウントを用意する。
- DLsiteで無料ゲーム登録が可能か、作品登録画面または問い合わせで確認する。
- Steamの30日待機をこの時点で確実に開始させる。

成果物:

- Steam App ID
- DLsiteサークル
- BOOTH/itch.io下書きページ
- DLsite無料登録可否メモ

### 2026-08-03〜2026-08-14: ストアページ下書きとSteamページ審査

- Steamストアページを完成させ、レビュー提出する。
- DLsite、BOOTH、itch.ioの作品ページ下書きを作る。
- Steamから差し戻しが来たら即日修正する。
- Steamページ承認後、Coming Soon公開の準備をする。

成果物:

- Steamストアページ承認
- 各サイト下書きページ

### 2026-08-17〜2026-08-28: Coming Soon公開と外部テスト

- Steam Coming Soonページを公開する。10月9日リリースなら、この時点で2週間要件は十分満たせる。
- BOOTH/itch.io限定公開URL、またはSteamテストキーで外部テストを始める。
- DLsiteページの予告登録が使えるなら申請する。
- SNS固定投稿、紹介動画、プレイ動画投稿ルールを準備する。

成果物:

- Steam Coming Soon公開
- 外部テスター向けビルド
- 宣伝導線

### 2026-08-31〜2026-09-11: リリース候補版RC1

- RC1を作成し、全ステージ通しテストを行う。
- SteamPipeへアップロードし、デポ/ブランチ/起動設定を確認する。
- DLsite/BOOTH/itch.io用ZIPも同じ内容で作る。
- 重大バグを修正し、RC2対象を絞る。

成果物:

- RC1ビルド
- SteamPipe設定
- サイト別ZIP

### 2026-09-14〜2026-09-18: Steamビルド審査提出

- Steamにほぼ最終ビルドをレビュー提出する。
- DLsiteに作品データチェック/審査を出す。無料ゲーム枠の都合で日付指定できない場合は予備日に回す。
- BOOTH/itch.ioは非公開状態で最終確認する。

成果物:

- Steamビルドレビュー提出
- DLsite審査提出
- BOOTH/itch.io最終下書き

### 2026-09-21〜2026-10-02: 差し戻し対応と最終凍結

- Steam/DLsiteの差し戻しに対応する。
- ストア文、スクリーンショット、年齢区分、権利表記の不一致を消す。
- 2026-09-25を目安にコンテンツを凍結する。
- 2026-10-02までにRC Finalを確定する。

成果物:

- Steam承認済みビルド
- DLsite承認または公開待ち
- RC Final

### 2026-10-05〜2026-10-09: リリース週

- 2026-10-05: 最終スモークテスト。起動、音、入力、終了、ZIP解凍、ウイルス誤検知チェック。
- 2026-10-06: プレス/告知文を予約投稿する。
- 2026-10-07: BOOTH/itch.io公開準備をロックする。
- 2026-10-08: Steam/DLsiteの公開時刻と手順を再確認する。
- 2026-10-09: Steamを手動リリースし、DLsite/BOOTH/itch.ioも同日公開する。

成果物:

- Steam公開
- DLsite公開
- BOOTH公開
- itch.io公開
- 告知投稿

### 2026-10-10〜2026-10-31: 初期運用

- クラッシュ、不具合、レビュー、コメントを毎日確認する。
- 重大バグのみホットフィックスする。
- FAQ、既知不具合、アップデート履歴を公開する。
- 10月末に配信結果を集計し、11月以降のアップデート方針を決める。
